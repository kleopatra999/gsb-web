Development - Helping the GSB Project

This page contains some useful information for people wanting to work developping the GSB source. Patches, comments, and suggestions are of course very welcome.

Mailing Lists ↑

There are several mailing lists used by the GNOME SlackBuild project. For more information, please see our Lists web page.

If you are a developer who would like to work on the GNOME SlackBuild project, you are encouraged to subscribe to two important mailing lists:

  1. GSB Development Discussion List
  2. GSB Code Commit List

ChangeLogs ↑

We have a variety of ChangeLogs available, include an SVN ChangeLog which gives an up to the minute description of the changes that are being made to the SVN developement tree, as well as ChangeLogs available for the binary builds of GNOME SlackBuilds. For a complete list of our ChangeLogs, please see ChangeLog web page.

Getting the code from Subversion: ↑

The only method supported of getting the GNOME SlackBuild tree is to check out our source code from our svn repository. Please make sure to do so in a directory on a disk partition that has quite a bit of free space. We have not yet unveiled a formal release version, so the most up-to-date build tree is found in trunk. You can check out the GSB Build System using the following command. 'trunk' holds our latest build tree and sources. This is where the majority of the work goes on for the upcoming stable release. At the moment, trunk is quite stable and ready for testing by the general public. When we are satisfied that we have a stable release, we will create a stable branch. These will be even numbered, similar to the GNOME versioning format. 

$ svn co http://svn.gnomeslackbuild.org/gsb/trunk gsb

You can also browse the SVN source tree via a web browser:

<http://svn.gnomeslackbuild.org/viewvc/>

Information about using subversion can be found in the Subversion Book.

What You Can Do To Help ↑

The best way to help the project is to use GSB. :^) With more users and testers, we are more likely able to catch and fix problems. Making sure to report problems you come across is very appreciated and helps immensely. The only thing greater than reporting a problem is offering a solution! Patches and fixes for GSB are welcome, and the easiest and preferred method is to use the GSB Development List highlighted above.

  • The Small Things Matter
    • Sometimes the package description file, slack-desc, may have improper spelling, or bad grammar. If you spot something, let us know.
    • slack-required files list all the dependencies of each package. If you find that a package you installed is missing a dependency, please point it out. These small things are sometimes accidently overlooked.
  • Hosting
    • MIRRORS are always needed, and the more the better. If you can help with hosting and setting up slapt-get repositories, please speak up! :)
  • Translations
    • Our web page and slack-desc files are only in English. If you are interested in helping translate the web page into other languages please let us know.
  • Testers
    • Binary Package Testers will be useful to catch any errors or installation problems with the packages we ourselves provide.
    • Build Tree testers are always welcome. Many individuals will prefer the "Build-It-Yourself" method, and we look forward to comments and patches from these users.
  • Graphics
    • Good graphics artists are always in high demand! If you're interested in designing icons, splash screen, gdm theme, backgrounds, or graphics for the web site, please don't hesitate to send us an email. We'd be very interested in hearing from you!
  • Suggestions
    • Do you think we're missing something every user of GNOME should have? Have a way that GSB can be better? Suggestions are welcome! Come along to chat in the #gsb IRC channel or send an email to the development list.

Our Roadmap ↑

What's the current long term plan for GSB? GNOME is under constant development and improvement, and one of our goals is to follow it as it progresses. Alongside the current stable version of GNOME, we will try to maintain a -current branch, which will house the latest and greatest GNOME development versions. Below is a brief outline of what we have planned.

  • Gnome Unstable Series
    • The GNOME Developers are currently working on GNOME , an unstable series which will eventually turn into a new 2.22 stable branch in about 6 months. Once our work is finished with GNOME , we will create a stable branch, and start work on a GSB-current branch, maintaining a bleeding-edge GSB series for all the hardcore GNOME enthusiasts.
  • Gnome Stable Series
    • Designed to be built and used on Slackware
    • Binary downloads are currently not available, but should be in a few days. Please keep an eye on the downloads page for any changes.
    • The Build system is quite stable now, and the majority of the work going on at the moment is bug squashing, and small changes to individual packages fixing the odd quirk. If you just can't wait for the binary distribution, give it a try! :)
  • Gnome 2.16.x -- Finished
  • Gnome 2.14.x -- Finished
